# auExpert i18n Codemap

**Last Updated:** 2026-03-31
**Status:** Complete migration to i18n (ZERO hardcoded strings)

---

## Overview

O sistema de internacionalização garante que **NENHUMA string visível ao tutor está hardcoded no código**. Toda mensagem, label, placeholder, erro e feedback é uma chave i18n.

**Por quê?**
- Mantém consistência entre idiomas (PT-BR, EN-US, futuro: ES-ES, FR-FR, etc)
- Permite tradução sem tocar código
- Força tom de voz consistente (voz do pet)
- Facilita A/B testing de mensagens

---

## Arquivos

### `i18n/pt-BR.json` (Portuguese)
~1400 chaves estruturadas em seções.

### `i18n/en-US.json` (English)
Espelho de pt-BR.json com traduções.

### `i18n/index.ts`
```typescript
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import ptBR from './pt-BR.json';
import enUS from './en-US.json';

i18n
  .use(initReactI18next)
  .init({
    fallbackLng: 'pt-BR',
    resources: {
      'pt-BR': { translation: ptBR },
      'en-US': { translation: enUS },
    },
    interpolation: { escapeValue: false },
  });

export default i18n;
```

## Uso em Componentes

### Function Component (Hook)

```typescript
import { useTranslation } from 'react-i18next';

export default function PetScreen() {
  const { t, i18n } = useTranslation();

  return (
    <View>
      <Text>{t('pets.myPets')}</Text>

      <Input
        placeholder={t('addPet.placeholderWeight')}
        label={t('addPet.weight')}
      />

      <Text>{t('auth.errorRequired')}</Text>

      {i18n.language === 'en-US' && (
        <Text>English-only content</Text>
      )}
    </View>
  );
}
```

### Class Component (sem hooks)

```typescript
import { useTranslation } from 'react-i18next';

class ErrorBoundary extends React.Component {
  render() {
    const { t } = this.context;

    return (
      <View>
        <Text>{t('errors.unexpectedTitle')}</Text>
      </View>
    );
  }
}

ErrorBoundary.contextType = I18nContext;
```

## Boas Práticas

### 1. Nomes de Chaves Semânticas

```typescript
// ✅ CERTO — semântico, hierárquico
t('auth.errorPasswordMin')
t('diary.helpVoice')
t('errors.network')

// ❌ ERRADO — genérico, não reutilizável
t('error1')
t('message2')
t('string_xyz')
```

### 2. Evitar Concatenação

```typescript
// ❌ ERRADO
<Text>{t('auth.email')} *</Text>

// ✅ CERTO
<Text>{t('auth.emailRequired')}</Text>  // ou adicionar chave `*` ao JSON

// ✅ COM INTERPOLAÇÃO
{ "auth.emailRequired": "{{label}} é obrigatório" }
t('auth.emailRequired', { label: t('auth.email') })
```

## Validação de Chaves

**Script para verificar chaves faltantes** (futuro CI/CD):

```bash
# Verificar se pt-BR.json tem todas as chaves de en-US.json
node scripts/validate-i18n.js
```

```typescript
// scripts/validate-i18n.ts
import ptBR from '../i18n/pt-BR.json';
import enUS from '../i18n/en-US.json';

function compareKeys(obj1, obj2, path = '') {
  const missing = [];
  Object.keys(obj2).forEach((key) => {
    const fullKey = path ? `${path}.${key}` : key;
    if (!(key in obj1)) {
      missing.push(fullKey);
    } else if (typeof obj2[key] === 'object') {
      missing.push(...compareKeys(obj1[key] || {}, obj2[key], fullKey));
    }
  });
  return missing;
}

const missing = compareKeys(ptBR, enUS);
if (missing.length > 0) {
  console.error('Chaves faltantes em pt-BR.json:', missing);
  process.exit(1);
}
```
